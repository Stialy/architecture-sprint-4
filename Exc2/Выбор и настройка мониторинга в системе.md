## Выбор подхода к мониторингу

### 1. MES API
Мониторить хочется много, начну с USE для выявления общих проблем с производительностью.
Далее расширять мониторинг в зависимости от полученных результатов

Подход USE
**Utilization** — Скорость обработки запросов API, обработки сообщений из очереди
**Saturation** — Утилизацию RAM, CPU, дисков, количество процессов обработки
**Errors** — Количество ошибок, возникающих при работе модуля

### 2. 3d file storage
Подход USE:

**Utilization** — Скорость обработки запросов на загрузку/скачивание файлов, размер S3.
**Saturation** — Утилизацию RAM, CPU, дисков, скорость репликации
**Errors** — Количество ошибок

### 3. Message Queue
Подход RED:

**Requests Rate** - Частота запросов
**Errors** - Количество неудачных запросов
**Duration** - Количество времени, которое уходит на обработку одного запроса

### 4. MES Database
    
Подход USE:

**Utilization** — Время обработки запросов, количество соединений, количество блокировок <br/>
**Saturation** — Ресурсы инстанса базы данных: потребление RAM, утилизацию CPU и дисков<br/>
**Errors** — Количество ошибок при работе с базой данных
